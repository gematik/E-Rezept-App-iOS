PharmacyLocation

Represents all information needed for searching for pharmacies.

  • init(id:status:telematikID:created:name:types:position:address:telecom:lastUsed:isFavorite:imagePath:countUsage:hoursOfOperation:avsEndpoints:avsCertificates:)

    Pharmacy default initializer

FHIR resources

  • id

    Id of the FHIR Location

  • status

    LocationStatus NOTE: Is here used to indicate E-Rezept readiness

  • telematikID

    Identifier of the pharmacy

  • created

    date of local client creation

  • name

    Name of pharmacy

  • types

    A pharmacy can have multiple types. In FHIR the code are e.g. “PHARM” and “OUTPHARM” and “MOBL”

  • position

    Position, i.e. Latitude and Longitude of the pharmacy’s address

  • address

    Address

  • telecom

    Telecom

  • lastUsed

    Bate of latest use for redeeming

  • isFavorite

    Bool indicating if user has marked this pharmacy as favorite

  • imagePath

    Path to an image of the pharmacy

  • countUsage

    Number of times this pharmacy has been used for redeeming

  • hoursOfOperation

    HoursOfOperation (opening hours)

  • avsEndpoints

    Container that holds urls to the AVS Endpoints and their certificates to send requests with the AVSModul

  • avsCertificates

    Array of certificates for all recipients

  • canBeDisplayedInMap
  • isErxReady
  • hasDeliveryService

    Indicates if the delivery service via the eRpRemoteStorage module (Fachdienst) is present Note: Authentication via “Fachdienst” is required

  • hasShipmentService

    Indicates if the shipment service via the eRpRemoteStorage module (Fachdienst) is present Note: Authentication via “Fachdienst” is required

  • hasReservationService

    Indicates if the reservation/onPremise service via the eRpRemoteStorage module (Fachdienst) is present Note: Authentication via “Fachdienst” is required

  • hasEmergencyService

    Indicates if the emergency service via the eRpRemoteStorage module (Fachdienst) is present Note: Authentication via “Fachdienst” is required

  • hasDeliveryAVSService

    Indicates if the delivery service via the AVS module (ApothekenVerwaltunsSystem) is present Note: No authentication via “Fachdienst” is required

  • hasShipmentAVSService

    Indicates if the shipment service via the AVS module (ApothekenVerwaltunsSystem) is present Note: No authentication via “Fachdienst” is required

  • hasReservationAVSService

    Indicates if the reservation/onPremise service via the AVS module (ApothekenVerwaltunsSystem) is present Note: No authentication via “Fachdienst” is required
